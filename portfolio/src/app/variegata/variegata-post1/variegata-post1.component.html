Stakeholder Information

While any stakeholders in the entertainment industry could benefit from research such as this, I am gearing it more toward video game designers. Video games, often more so than other story-based media, requires a less-than-linear story to be available to the consumer. As a result, designers of these video games would have a greater need for a tool to help keep those stories interesting and engaging for the player.

People who love video games, myself included, value an intricate plot that can change depending on the player's choices. This interactivity leads to re-playability, making the game more valuable in the eyes of the customer. Putting funds, designers' knowledge, and ample technology into research to create increasingly intriguing stories benefits all parties involved.

Existing Research

The following are some relevant papers from previous research that I hope will guide this project.

  • US5805784A - Computer story generation system and method using network of re-usable substories

    Patent for a method of generating a story based on smaller substories. This method breaks a substory consisting of a few sentences into a matrix defining features such as who is involved, what specific topics they are saying, what specific events are happening, etc. This matrix allows characters and scenarios to be reusable, scenarios to be seamlessly transitioned between, and chosen by the algorithm on the fly during user-interaction.

  • Story Generation with Crowdsourced Plot Graphs

    This paper uses a crowdsourced corpus of narrative examples of a new domain, a model can be automatically constructed to allow for samples to be taken based on desired criteria. A plot graph can then be constructed using this model to show the potential flow of the story. This graph includes required and optional events, events that strictly rely on one another, and events that must be mutually exclusive. Using this graph allows for a generated story from beginning to end that has relatively few inconsistencies.

  • A Character-Centric Neural Model for Automated Story Generation

    This paper focuses more on using a predefined character, context, and action to generate an event. The researchers compare quality of events generated with and without using these predefined traits as input. It was found that providing the characters-centric information from the beginning significantly increases a story's consistency and explainability.

  • Event Representations for Automated Story Generation with Deep Neural Nets

    Using neural nets, these researchers use methods for converting basic information of an event into a sentence and vice versa. Starting with the original sentences, the method breaks them down in multiple steps, calculating a measurement for perplexity at each stage. It was found that they were able to automatically break the input down into general elements and significantly reduce the perplexity, allowing the generated event to be built up once again into new sentences to be added to the story.

  • Learning to Tell Tales: A Data-driven Approach to Story Generation

    This paper takes a look at breaking down an input sentence or story and restructuring the various elements into a tree that can be traversed to create a new story. This allows a single data structure that can be reused to generate new scenarios each time depending on the path taken during the traversal. This method was found to produce stories that lack much variation from one another, and rather boring plots.
