Debugging Limbo

This week seemed to be a never-ending cycle of issues. When trying to train the model on a larger dataset using SQLite as the database system, my computer kept having troubles handling the larger files. Additionally, uploading the database to my web hosting server performed even worse. I then made the move to my web host's integrated database system - MariaDB (a fork of MySQL).

The issue I ran into next was getting access to the database from my python scripts. The web host does not allow direct remote access to the database, so I had to figure out how to set up an SSH tunnel to send the requests from my local machine to the server. After a bit of debugging and reading up on the documentation for the MariaDB python module, I was able to get it working on my computer.

Just when I thought everything was fine and dandy, I pushed the update to the web host server. Of course, everything broke once again. This time my entire site went down, giving only a 500 error. At this point I contacted my web host via live chat, and went through probably the longest tech support session I've experienced. When we figured out that the 500 error was due to a single malformed file, we found that the MariaDB module in python does not work properly with the server due to certain connector packages not being available, which I found odd considering this is the recommended way to manage their built-in databases through python. Many hours and three tech support specialists later, no solution was found. They finally decided to pass it on to their technicians and opened up a ticket, which I am still waiting to be dealt with on their end.

Long story short, not much progress was made on the actual functionality of the app. I can just hope that it all gets resolved soon.
